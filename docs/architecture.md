# Architecture

`twave-knowledge-atlas` is a local read model built from gateway exports.

It has three layers:

1. Gateway export ingestion: reads JSONL or JSON files generated by TWave gateways.
2. Local atlas store: keeps entities, aliases, relations, source references, update logs, and FTS search in SQLite.
3. Consumer interfaces: CLI, Obsidian vault, and stable DB tables for future `twave-agent-hub` integration.

The atlas never calls Odoo, Notion, SGC, or research gateways directly. Gateways remain the source of detailed truth. The atlas stores summaries, relationships, and routes.

## Data Flow

```text
gateway export -> atlas import -> SQLite -> merge -> vault/search/routes
```

## Agent-Hub Usage

`twave-agent-hub` can query:

- `entities` and `aliases` to resolve a user mention.
- `relations` to expand context.
- `source_refs`, `known_in`, and `best_tools` to decide which gateway/tool should answer detail questions.
- `search_index` for lightweight local discovery.
